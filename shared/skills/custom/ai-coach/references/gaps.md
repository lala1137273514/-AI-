# 缺口 / 待孵化清单（答不上来的需求 = 冲刺选题池）

> 条目格式（新增照抄）：
> ```
> ## [G-编号] <需求一句话>
> - 日期：YYYY-MM-DD    提出人：<谁>    被问次数：1
> - 需求详情：要解决什么、输入输出是什么
> - 最近的现有能力：<registry 里最接近的条目>，差在哪
> - 状态：open | in-progress(@认领人) | closed(已入注册表)
> ```
> 规则：只追加和改状态/计数；同类需求再被问到 → 「被问次数 +1」，≥2 触发沉淀督促；closed 不删行。
