## 一、反推:为达成该 KR,每周应做什么 + 应有产出

判定核心是**"稳定 / 习惯"**(贯穿 Q2 的持续性),三条腿:① 基础需求整理(持续输出需求文档)② 沟通对齐(持续参与/主导评审、对齐会)③ 文档输出习惯(日报/周报/月报日更周更)。本 KR 看的不是单点成果,而是**节奏的稳定性**。

| 周 | 应做什么 | 应有产出 |
|---|---|---|
| 全 Q2 每日 | 日复盘 / 日报 | 每日 1 篇日复盘(连续不断档) |
| 全 Q2 每周 | 周报 + 周回顾 | 每周 1 篇周报 |
| W1~W6 | 持续承接并整理需求、过评审 | 需求文档 + 评审/对齐纪要持续产出 |
| W1~W6 | 跨团队沟通对齐(需求/版本/测试/方案会) | 多场对齐会纪要 + 共识结论 |
| 习惯化 | 把复盘 / 周报 / 对齐自动化、固化为流程 | daily-review skill、cron 定时、复盘日历 |

**判定标准**:日报 / 周报覆盖率高且连续;评审与跨团队对齐贯穿全程;需求整理→沟通→文档三件套形成**可重复的稳定节奏**,且能**较独立**完成。

- **Gap(非"没做",是节奏与可见性细节)**:
  1. 个别工作日日报并入周报未单独成篇(5/5、5/7、5/16、5/17 等),不影响"稳定"判定,但若严格要求"日更连续"可补齐归档。

  3. 自动化/复盘工具多在本机 skill + 飞书复盘文件夹,leader 对"自动化程度"的可见性可在进展记录中点明。
