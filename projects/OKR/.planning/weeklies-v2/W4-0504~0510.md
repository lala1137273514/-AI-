## 💡 核心洞察

1. **本周的主线不是某个交付物,而是角色升级。** 5/6 从需求拆解者变成「用户记忆与个性化」专项 Owner——O2-KR1 不再是「识别 ≥3 个场景」的练习,而是对一条方向的场景/需求/落地负责。当天记忆三层 PRD 即承接落地,职责与产出同日对上。
2. **Actor/Judge/Optimizer 是本周最贵的方法资产,Kickoff 把它组织化。** 5/7 提出三角色进化闭环解「风格难收敛、只能加 few-shot」,5/9 Kickoff 收拢为「闭环迭代 + 多 session 并行 + 人性化评测接 Langfuse」——从个人探索到有组织推进,且「评测接 Langfuse」直接锚定了 W6 rowboat 评测器用真 trace 跑通的数据飞轮。
3. **「有条件通过」与「203 case/69 失败」是同一种工程严谨。** 530 审查不放行、抓致命矛盾;测试会不只数失败、定断言分层与评估器权重——需求质量与测试判分都从手感升级为标尺,这是 Owner 角色把方向做扎实的底色。
4. **W4 是被旧侧记漏得最多的一周。** 旧侧记主要落在会议与 Slack——本版据本机 G 盘 + Claude/Codex 会话**补回了 530 三线 MVP 6 份 PRD + Actor/Judge/Optimizer 进化闭环 + Agent 个性化专项 + 430 文本生成存放歧义方案**四条工程/方法线,工程实质远比旧侧记厚。
