## 🧠 个人思考（本周反思）

1. **文档生成需求绕了几圈才想通**。一开始默认要绑谷歌全家桶，还纠结"飞书同生态、谷歌不是"，后来反应过来 Loona 本就面向海外、走谷歌没问题；真正解套是发现产物存储不依赖谷歌硬盘、谷歌文档本身就能存——于是把"建一套文档系统"收窄成"只打通生成链路、MVP 验证"。需求大的时候，先找能砍的边界。
2. **越来越清楚自己是 PM 不是技术**。调研完谷歌 API 我反而踩了刹车：PRD 不该把所有 API 字段标死，我给需求和边界，字段实现是技术那边的事。这周一直在校准自己产出的颗粒度和角色边界。
3. **旅行规划这条线我反复推翻自己**，根子是"卡片太满、太程序化"这个老板痛点没解透。最后收敛成三轮澄清→大图方案三选一→钻取详情，但 plan 和 trip 怎么解耦、先调谁，当时没拍死，是带着疑问往下走的——这块还欠一个干净的架构决定。
4. **方法论上有个明确升级：能并行别串行、能用 skill 别手搓**。这不是被动执行，是看了一周自己的工作流之后主动沉淀，还立刻用上了——把旅行链路封成 skill、并行跑子任务。
5. **对汇报的态度变了**。不太信"嘴上说的成果"，要用真实 session + commit 反推，文字还要去 AI 味再自审一遍才发。这周这份周报就是这么做的。
