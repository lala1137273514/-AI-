## 一、背景与范围

### 1.1 分析对象

本次系统体验分析的对象是 **Loona Deskmate 现有 Agent 能力链路**，核心为两层结构：

- **Router（路由/意图层）**：判断用户输入的意图方向（CHAT / NEW / EXISTING / NO / NA），决定是否承接已有任务、是否调用工具。"方向判得对不对"。
- **Planner（执行/规划层）**：在 Router 判定后，选择工具、抽取参数、组织执行步骤并生成回复。"动作做得对不对"。

围绕这两层，分析覆盖的能力维度包括：**对话/意图理解、任务执行与闭环、工具调用、短时记忆与状态一致性、上下文承接（多轮）、相对时间理解、回复质量与人性化收束**。

## 二、Agent 能力系统体验分析（按能力维度）

> 方法：以 Router/Planner 两层链路为骨架，逐维度梳理"现状—体验短板—证据"。指标口径以《MVP 评测考察场景》第五章 8 个正式指标为准。

### 2.1 意图理解与路由（Router 层）

- **现状**：Router 负责 CHAT/NEW/EXISTING/NO/NA 五类意图分发，核心指标 `intent_accuracy`、`task_link_accuracy`、`router_tool_list_accuracy`（弱包含判定）。
- **体验短板**：意图误判直接导致后续链路走错；EXISTING（承接已有任务）识别失败是上下文断裂的主要根因（见 2.5）。Router 工具列表采用"弱包含"判定——实际多给工具算通过，少给或写错算故障，宽松度高于 Planner。
- **证据**：《MVP 评测考察场景》§2.2、§5.2；4/9 系统问题定位会议指出"复杂任务缺过程反馈"拟通过"加重 Router 职责产生第一步反馈"解决。

### 2.2 任务执行与闭环（Planner 层）

- **现状**：Planner 负责工具选择（`tool_selection_accuracy`，强匹配）、参数抽取（`param_accuracy`，强匹配，不能错/多/少）、回复生成（`planner_reply_accuracy`）。指标体系以"用户任务闭环成功率"为北极星，强调"做对、做完、做顺"三位一体。
- **体验短板**：闭环不完整——存在"伪完成"（接口返回成功但实际未达成目的）、未确认即执行、过程缺反馈。死循环阻塞：任务卡在某条链路出不来，且后续任何输入都失效（见核心问题 #2）。
- **证据**：`Agent 相关指标.md` 设计原则 §6（重点覆盖"结果错误或伪完成、未确认即执行"）；4/9 会议"决策机在执行某些素材时退不出来，卡在链里"。

### 2.3 工具调用

- **现状**：接口测试报告 217 用例，通过率 97.24%（211 通过 / 5 失败 / 1 异常）。失败集中在 Planner 工具名称识别与参数断言。
- **体验短板**：工具名称识别偏差（预期调用某工具但实际未调用或调用了其他）、工具参数断言不一致（已调用但入参与预期不符）、空值处理不足（预期为 None 时触发 TypeError，边界健壮性不足）。
- **证据**：接口测试报告 §4–§5；典型失败 PLANNER-067（预期 `get_mail_list` 实际空列表）、PLANNER-081（预期空但实际调用 `list_contacts`）、PLANNER-090（`NoneType` 类型异常）。

### 2.4 短时记忆与状态一致性

- **现状**：评测痛点二"大模型幻觉专项压测"明确要求参数与回复"完全映射并基于最新真实对话状态"。
- **体验短板**：用户连续修改会议主题/时间/地点/参会人后，模型仍引用旧参数、回复混入已废弃信息（脏数据残留）；状态展示不一致。
- **证据**：《MVP 评测考察场景》§3.2 痛点二（场景 3 短时记忆最新状态留存）；指标 `param_accuracy`、`planner_reply_accuracy`。

### 2.5 上下文承接（多轮）

- **现状**：痛点三"上下文承接出错与链条阻断"，指标 `continuation_decision_accuracy`（落地为 `task_link_accuracy`）。单/多轮区分不取决于对话条数，而取决于当前步骤是否依赖历史上下文。
- **体验短板**：先查邮件/日程再说"回复这封""把它删了"，用户未重复指明对象时系统却要求重新澄清；Router 未识别为 EXISTING 导致链路断裂；Planner 未正确继承前序对象 ID。
- **证据**：《MVP 评测考察场景》§3.2 痛点三（场景 5 跨意图隐式承接）；测试集专项会三痛点之一。

### 2.6 相对时间理解

- **现状**：痛点二场景 4，指标 `param_accuracy`。
- **体验短板**："后天/下周二/下下周/月底"等相对时间理解错误；跨周/跨月/跨日期边界计算偏移；时间槽位抽取正确但标准化结果错误。
- **证据**：《MVP 评测考察场景》§3.2 场景 4。

### 2.7 回复质量与人性化收束（W6 延伸佐证）

- **现状**：5 月下旬独立建成"人性化收束层评测器"，度量把 LLM 原始输出改写得像人说话、同时保住人格、不损坏事实。
- **体验短板**：人性化改写存在**事实损坏**——生产 humanizer（qwen-plus）编造事实，约 **23% evidence_diff FAIL**（该数判官无关，稳定）。
- **证据**：[人性化收束层评测器成果展示](https://f4x6dn8llc.feishu.cn/docx/CdnJdNJlyo7tvDxiOMBcR1fbnoc)（benchmark 112 条：5 scene × 7 scenario × 3 persona + 13 对抗边界）。

## 三、核心问题清单

> 严重度：P0 致命/阻断；P1 高影响；P2 中等；P3 体验/规范类。证据带链接或本机路径。

### 问题 #1 — 重复回复与冗余播报

- **描述**：面对脏数据、找不到资源、不支持的意图时，系统反复尝试执行、反复播报相同内容，未做顺畅兜底拦截。如"联系人不存在仍反复尝试执行"。
- **证据**：《MVP 评测考察场景》§3.2 痛点一场景 1/2；4/9 系统问题定位会议；测试集专项会三痛点之一。
- **影响**：用户感知到 Agent"卡壳/复读"，体验极差，破坏对话自然度。
- **严重度**：**P1**

### 问题 #2 — 死循环阻塞 / 任务卡链不可恢复

- **描述**：任务卡在某条执行链路出不来（业务层卡 + 决策机卡两类），状态写错导致卡死，且"后续你再跟他说任何东西都没有用了"。
- **证据**：4/9《Agent 任务与能力系统问题定位》纪要；4/1 决策机通讯架构梳理（任务链切换死板等 4 个系统关键问题）。
- **影响**：会话整体失效，用户需重启，闭环成功率归零。
- **严重度**：**P0**

### 问题 #3 — 大模型幻觉 / 旧状态污染

- **描述**：用户连续修改实体（会议主题/时间/参会人）后，模型仍引用旧参数、回复混入已废弃信息；自行推演补全产生捏造内容。
- **证据**：《MVP 评测考察场景》痛点二场景 3；`Agent 相关指标.md` §6。
- **影响**：执行结果错误（如按旧时间建日程），可信度受损。
- **严重度**：**P0**

### 问题 #4 — 上下文承接不足 / 链条断裂

- **描述**：多轮场景中 Router 未识别为 EXISTING，或 Planner 未继承前序对象 ID，导致"回复这封邮件/把它删了"等隐式承接失败，要求用户重新澄清。
- **证据**：《MVP 评测考察场景》痛点三场景 5；测试集专项会三痛点之一。
- **影响**：多轮任务断裂，用户被迫重复表达，体验割裂。
- **严重度**：**P1**

### 问题 #5 — 相近资源串台 / 属性污染

- **描述**：两个相似会议/邮件/联系人混淆，修改 A 误操作到 B，不同实体属性串用。
- **证据**：《MVP 评测考察场景》痛点三场景 6；指标 `tool_selection_accuracy`、`param_accuracy`。
- **影响**：误操作真实资源，可能造成不可逆后果。
- **严重度**：**P1**

### 问题 #6 — 工具调用偏差（名称/参数/空值）

- **描述**：Planner 工具名称识别偏差、参数断言不一致、空值处理不足。217 用例中 5 失败 1 异常集中于此。
- **证据**：接口测试报告 §4–§5（PLANNER-067/081/090）。
- **影响**：动作结果不准确、业务一致性受损；空值场景直接抛 TypeError。
- **严重度**：**P1**

### 问题 #7 — 复杂任务缺过程反馈

- **描述**：复杂任务执行期间缺少中间过程反馈，用户不知系统在做什么，疑似卡死。
- **证据**：4/9 系统问题定位会议第一条；拟通过加重 Router 职责产生第一步反馈解决。
- **影响**：过程体验差，用户焦虑/误判为故障。
- **严重度**：**P2**

### 问题 #8 — 测试产出与 PRD 不一致（11 项）

- **描述**：测试团队实际产出（接口测试报告 + 自动化平台技术方案）与 PRD（《MVP 评测考察场景》）存在 11 项不一致，集中体现为：① 场景标签缺失（无 `scenario_tag` 归类）；② 指标口径不对（用"工具名称断言"等笼统描述，未对标 8 个正式指标）；③ 聚合视图缺失（无 req_id→trace_id→case_id 两级聚合）；④ 失败归因不够（未按 5 层归因）；⑤ 用例编号不规范；⑥ 样本范围未对齐（跑 217 条而非定向 20 条基准）；⑦–⑪ 技术方案侧标签术语另造三类、阶段过度拆分、断言法则未固化、聚合口径表述歧义、MVP 节奏错配（过度平台化）。
- **证据**：`测试产出与 PRD 不一致汇总.md`（一/二两节，共 11 项）。
- **影响**：badcase 无法结构化沉淀、统计口径无法统一、无法从 trace/case 视角看链路成败、MVP 资源被平台化建设挤占。
- **严重度**：**P1**（流程/可沉淀性硬伤）

### 问题 #9 — 长链路延时

- **描述**：接口测试平均用例耗时 3150ms，复杂多工具链路延时叠加，影响"做顺"的过程体验。
- **证据**：接口测试报告 §1（217 用例总耗时 12 分 51 秒，平均 3150ms）。
- **影响**：响应迟缓，过程体验下降；指标体系将时延列为体验型补充指标。
- **严重度**：**P2**

### 问题 #10 — 人性化收束事实损坏 ≈ 23%（W6 延伸佐证）

- **描述**：人性化改写层（humanizer，qwen-plus）在改写时编造/篡改事实，约 23% evidence_diff FAIL。
- **证据**：[人性化收束层评测器成果展示](https://f4x6dn8llc.feishu.cn/docx/CdnJdNJlyo7tvDxiOMBcR1fbnoc)（判官无关的确定性事实门测得，稳定）。
- **影响**：用户看到的"更像人"的回复可能事实错误，信任受损；揭示"像人"与"保真"的张力。
- **严重度**：**P0**（事实正确性）

### 问题 #11 — Cortex 链路 task 失败率 6.25%（W6 延伸佐证）

- **描述**：5/20 Cortex 深度问题报告测得 task 失败约 6.25%。
- **证据**：Jarvis session（Cortex 5/20 深度问题报告）；ledger `claude.md` 2026-05-21 行。
- **影响**：真实链路仍有可观失败率，需持续回归。
- **严重度**：**P1**

## 四、指标与评估视角

指标体系（`Agent 相关指标.md` + 《MVP 评测考察场景》§5）覆盖**用户可直接感知**的四类维度，不覆盖底层模型内部/单工具实现/纯工程过程指标：

| 维度 | 关注 | 正式指标 |
|---|---|---|
| **可感知任务结果**（做对） | 意图判断、路由、工具选择、参数正确性、结果正确性 | `intent_accuracy`、`task_link_accuracy`、`router_tool_list_accuracy`、`tool_selection_accuracy`、`param_accuracy` |
| **过程体验**（做顺） | 时延、状态一致性、稳定性、过程反馈 | 时延（体验型补充）、状态一致性 |
| **对话质量**（说得对） | 回复准确性/完整性/边界解释/是否幻觉 | `router_reply_accuracy`、`planner_reply_accuracy` |
| **工具调用必要性**（做该做的） | 该调不调/不该调却调 | `tool_policy_accuracy`（跨表派生） |

**判定与聚合规则**：
- **断言强度分层**：Router 工具列表用**弱包含**（实际含预期即通过）；Planner 工具/参数用**强匹配**（不能错/多/少）。
- **两级聚合**：req_id→`trace_id`（一处错则全错）→`case_id`（Session 级，任一 trace 失败则整 case 失败），为 badcase 追踪提供基础。
- **5 层失败归因**：Router 意图层 / 续接层 / 工具选择层 / 参数抽取层 / 回复生成层。
- **北极星**：用户任务闭环成功率（在可接受时延内正确完成闭环）。
- **W6 评估方法论延伸**：两层评测（确定性指标 + LLM-as-Judge），评委可信度需校准——盲评 Claude 100% 复现作者标注，qwen-plus 仅 84.3%（κ=0.727），结论"事实门零方差可证、人格评委质量上限=评委模型本身"。

## 五、结论与下一步建议

### 5.1 结论

- 4 月内**系统体验分析的过程证据充分**：建成评测集与标注规范（v1→v2.0）、Langfuse 可观测设计、Agent 可感知指标体系，围绕**重复回复 / 幻觉 / 上下文承接**三痛点泛化 6 场景，产出接口测试报告（217 用例 / 97.24%）与"测试产出与 PRD 不一致汇总"（11 项）。
- **形上有缺口**：4 月内这些素材散落在多份文档与周会纪要中，**未收口为单一署名的"问题分析文档"**——本文即为该收口物，但成文于 5/24，**逾期 24 天**。
- 共梳理 **11 条核心问题**：4 月内 9 条（#1–#9）+ W6 延伸佐证 2 条（#10–#11）。其中 P0 级 3 条（死循环阻塞、幻觉污染、人性化事实损坏）。
