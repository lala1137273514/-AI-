## 4. 复盘洞察（思考）

1. **「Loona 是 Copilot 的语音化输出适配器,不是新 agent」是今天最贵的判断。** 这一句话把工程范围从「再造一个 agent」收敛为「只在输出侧做语音化适配,不动编排层」——省掉了一整条可能走偏的重活,也直接决定了 DashScope TTS 接入的落点。一个准确的产品定位胜过十次乱写代码。
2. **三源在同一天罕见地同时命中,且互为闭环。** Claude 出 TTS 接入 + 适配器 insight、Codex 出 6 wave 多 agent 编排、G 盘出 Agent-UI 通用范式——语音(TTS)、编排(wave)、界面(UI 范式)三条线同日成形,正好覆盖 Loona「听得见、调得动、看得懂」三个面。
3. **「4877 run 里仅 39 copilot、99.2% 浪费」是一条值得记下的性能账。** 把性能问题量化到具体数字,后续清理才有靶子;这种「先量化再优化」的习惯,和评测器的确定性指标取向一脉相承。
